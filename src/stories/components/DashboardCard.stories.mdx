# DashboardCards

### DashboardCards are components used to build cards with information that requires a short description and can be classified into status.
---
<br />

## When to use:
- It is necessary to show information in summary form on dashboards
- A card with a clickable button that takes the user to a new screen is required.
- The information described on the card can be classified.


<br />

## When not to use:
- There is a very large set of information to be displayed.
- If you want to use icons together with the data. Use <a href="https://rds-design-system.netlify.app/?path=/docs/componentes-containers-expansioncard--expansion-card"> ExpansionCards </a> in these scenarios.

<br />

## Comments:
- DashboardCard displays information vertically, unlike <a href="https://rds-design-system.netlify.app/?path=/docs/componentes-containers-expansioncard--expansion-card">ExpansionCard</a >.
- <a href="https://rds-design-system.netlify.app/?path=/docs/componentes-containers-expansioncard--expansion-card">ExpansionCards</a> communicate information more directly and with assistance of icons. DashboardCars focus more on description and classification of information.
- Consider using DashboardCards if you need the component to redirect the user to another page for more information.
- Consider <a href="https://rds-design-system.netlify.app/?path=/docs/componentes-containers-expansioncard--expansion-card">ExpansionCards</a> if you need to detail the card content , but without removing the user from the page.
