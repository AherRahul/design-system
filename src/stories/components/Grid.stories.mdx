# Layout Grid

### A flexible grid system for structuring and organizing content within web applications.

---

<br />

## When to use:

- **Structured Layouts:** Arrange content in a responsive grid layout.
- **Grid-based Designs:** Create consistent and visually appealing designs.
- **Responsive Design:** Adapt layouts across various screen sizes.
- **Component-Based Layouts:** Integrate seamlessly with other components.

<br />

## When not to use:

- **Non-structured Content:** When content doesn't need a predefined layout.
- **Complex Grid Needs:** For grids requiring highly customized layouts beyond basic column and row configurations.
- **Performance Considerations:** Overuse in scenarios where simpler CSS layouts suffice.

## Configuration

### Grid Structure

Initialize a layout grid with the `rds-grid` custom element, and then add rows, defined as `rds-row`, and columns within each row, defined as `rds-col`.

## Usage Example

### 1. col="3" offset="0"

<br />

<Story
    name="Grid1"
    args={{
      cols: 3,
      offset: 0,
      colsAtS: 12,
      colsAtM: 12,
      colsAtL: 4,
      colsAtXl: 4,
    }}
  >
    { Template.bind({}) }

```html
<rds-grid>
    <rds-row>
        <rds-col cols="4" offset="0" colsAtS="12" colsAtM="6" colsAtL="4" colsAtXl="4">1 of 3 Columns 30%</rds-col>
        <rds-col cols="4" offset="0" colsAtS="12" colsAtM="6" colsAtL="4" colsAtXl="4">2 of 3 Columns 30%</rds-col>
        <rds-col cols="4" offset="0" colsAtS="12" colsAtM="6" colsAtL="4" colsAtXl="4">3 of 3 Columns 30%</rds-col>
    </rds-row>
</rds-grid>
```

- `rds-row` elements contain columns.
- `rds-col` elements contain content (or other rows, see nesting) and must be immediate children of rows.
- By default, column widths are set in percentages, sized fluidly relative to their parent.
- Gutters (the space between columns) are included by adding padding to each column, resulting in space between columns.
